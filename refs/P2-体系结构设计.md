# Phase 2：体系结构设计

**周次：** 第 8-9 周 | **人均工时：** 6 小时 | **团队总工时：** 24 小时 | **提交时间：**2026年5月3日 23:59之前

---

## 一、本阶段目标

1. 选择合适的架构风格并给出合理性论证
2. 完成模块划分与接口定义
3. 编写架构决策记录（ADR）
4. 体验 AI 在架构设计中的辅助能力与局限

---

## 二、任务说明

### 任务 1：架构风格选择与论证（2 小时）

#### 1.1 候选架构分析
结合课堂所学的架构风格（MVC、分层、微服务、事件驱动等），分析至少 2 种候选架构方案，从以下维度进行对比：

| 对比维度 | 方案 A | 方案 B |
|---------|-------------|-------------|
| 开发复杂度 | | |
| 可维护性 | | |
| 可扩展性 | | |
| 性能 | | |
| 团队学习成本 | | |
| 10 周内可行性 | | |

#### 1.2 关键实验 —— "架构辩论赛"

**实验流程：**

1. **每位成员**分别用不同的 Prompt 策略向 AI 咨询架构建议：
   - 成员 A：直接问"推荐什么架构"
   - 成员 B：提供详细项目约束后问
   - 成员 C：要求 AI 对比多种方案的优劣
   - 成员 D：让 AI 扮演架构师角色进行分析

2. **记录每个 Prompt 和 AI 的回答**

3. **团队辩论会**（线下或线上，30 分钟）：
   - 每人展示 AI 给出的方案和理由
   - 讨论不同 Prompt 策略下 AI 回答的质量差异
   - 团队投票选择最终方案

4. **记录**：最终选择了什么，为什么？**为什么不选**其他方案？

### 任务 2：模块划分与技术选型（2 小时）

#### 2.1 模块划分
基于选定的架构风格，将系统划分为若干模块，明确：
- 每个模块的职责边界
- 模块间的依赖关系（绘制模块依赖图）
- 模块间的接口（调用方式、数据格式）

#### 2.2 技术选型
确定项目的技术栈，包括：

| 层次 | 选择 | 选择理由 |
|------|------|---------|
| 前端框架 | | |
| 后端框架 | | |
| 数据库 | | |
| ...                                           |      |          |
| 中间件1（含缓存、消息队列等，没有用到可不填） |      |          |
| 中间件2                                       |      |          |
| ...                                           |      |          |
| 部署方式                                      |      |          |

数据库可选择不止一个（如有需要），中间件请根据实际类型填写（如：缓存、消息队列、网关等）

### 任务 3：架构决策记录 ADR（1 小时）

编写至少 **3 条 ADR**（Architecture Decision Records），格式如下：

```
# ADR-001：[决策标题]

## 状态
已接受 / 已废弃 / 待讨论

## 背景
描述面临的问题和约束条件

## 决策
我们决定采用 ……

## 理由
为什么做出这个决策（列出考虑的因素和权衡）

## 后果
- 正面影响：……
- 负面影响：……
- 需要关注的风险：……

## AI 辅助记录
- AI 初稿内容摘要：……
- 人工修订内容：……
- 修订理由：……
```

**要求：** 每条 ADR 必须有 AI 初稿与人工修订的对比。重点审查 AI 是否遗漏了重要的质量属性权衡。

### 任务 4：架构文档编写（1 小时）

整合以上内容，形成完整的架构设计文档：
1. 架构概览（含架构图）
2. 模块划分说明
3. 技术选型说明
4. ADR 集合
