## 二、核心用例详细描述

### 2.1 用例 UC-03：发布跑腿任务

| 项目 | 内容 |
|------|------|
| **用例名称** | 发布跑腿任务 |
| **参与者** | 普通用户（已完成学号认证） |
| **前置条件** | 1. 用户已登录系统<br>2. 用户已完成学号实名认证<br>3. 用户账户有足够积分（如报酬 > 0） |
| **后置条件** | 1. 任务发布成功，出现在需求大厅<br>2. 用户积分被冻结（等待任务完成）<br>3. 任务状态为“待接单” |

**基本流：**

| 步骤 | 操作 |
|------|------|
| 1 | 用户进入“发布任务”页面 |
| 2 | 用户选择任务类型为“跑腿代取” |
| 3 | 用户填写任务信息：取件地点、送达地点、物品描述、报酬积分（可为0）、有效期、校区 |
| 4 | 用户确认发布 |
| 5 | 系统验证用户积分是否足够（如报酬 > 0） |
| 6 | 系统冻结相应积分（如报酬 > 0） |
| 7 | 系统生成任务，状态设为“待接单” |
| 8 | 任务出现在需求大厅列表 |
| 9 | 系统向用户返回发布成功提示 |

**替代流：**

| 步骤 | 分支 | 处理 |
|------|------|------|
| 3a | 用户填写信息不完整 | 系统提示“请填写完整信息”，不提交 |
| 5a | 用户积分不足（报酬 > 0 且积分 < 报酬） | 系统提示“积分不足，无法发布”，不冻结积分 |
| 6a | 积分冻结失败 | 系统提示“发布失败，请重试” |
| 3b | 用户点击取消 | 返回上一页，不保存信息 |

### 2.2 用例 UC-04：接单

| 项目 | 内容 |
|------|------|
| **用例名称** | 接单 |
| **参与者** | 普通用户（已完成学号认证） |
| **前置条件** | 1. 用户已登录系统<br>2. 用户已完成学号实名认证<br>3. 存在“待接单”状态的任务<br>4. 用户不是该任务的发布者 |
| **后置条件** | 1. 任务状态变更为“进行中”<br>2. 其他用户不可再接此单<br>3. 创建接单方与发布方的聊天会话 |

**基本流：**

| 步骤 | 操作 |
|------|------|
| 1 | 用户在需求大厅浏览任务列表 |
| 2 | 用户点击某个任务查看详情 |
| 3 | 用户点击“接单”按钮 |
| 4 | 系统检查任务状态是否为“待接单” |
| 5 | 系统将任务状态更新为“进行中” |
| 6 | 系统记录接单方信息 |
| 7 | 系统创建任务双方的聊天会话 |
| 8 | 系统向发布方推送“已有人接单”通知 |
| 9 | 系统向接单方返回“接单成功”提示 |

**替代流：**

| 步骤 | 分支 | 处理 |
|------|------|------|
| 4a | 任务已被其他人接走 | 系统提示“任务已被接走”，返回任务列表 |
| 4b | 任务已过期 | 系统提示“任务已过期”，返回任务列表 |
| 5a | 状态更新失败 | 系统提示“接单失败，请重试” |
| 3a | 用户是任务发布者 | 系统提示“不能接自己的任务”，禁止接单 |

### 2.3 用例 UC-14：互评

| 项目 | 内容 |
|------|------|
| **用例名称** | 互评 |
| **参与者** | 普通用户（已完成任务的双方） |
| **前置条件** | 1. 任务状态为“已完成”<br>2. 评价方尚未对该任务进行评价<br>3. 任务完成时间在 48 小时内 |
| **后置条件** | 1. 评价记录保存到数据库<br>2. 被评价方的好评率重新计算<br>3. 任务双方的评价记录公开可见 |

**基本流：**

| 步骤 | 操作 |
|------|------|
| 1 | 用户进入“我的任务”页面 |
| 2 | 用户点击“待评价”标签 |
| 3 | 用户选择某个已完成的任务 |
| 4 | 用户选择评价等级（好评/中评/差评） |
| 5 | 用户填写评价内容（差评需填写理由） |
| 6 | 用户提交评价 |
| 7 | 系统保存评价记录 |
| 8 | 系统更新被评价方的好评率 |
| 9 | 系统向被评价方推送“收到新评价”通知 |

**替代流：**

| 步骤 | 分支 | 处理 |
|------|------|------|
| 4a | 用户选择“差评”但未填写理由 | 系统提示“差评需填写理由”，不提交 |
| 5a | 用户提交后 | 评价不可修改（需通过申诉流程处理争议） |
| 2a | 用户在 48 小时内未评价 | 系统自动默认好评 |
| 6a | 评价提交失败 | 系统提示“评价失败，请重试” |
| 4b | 用户点击取消 | 返回上一页，不保存评价 |
