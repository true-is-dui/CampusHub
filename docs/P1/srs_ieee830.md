## 1. 引言

### 1.1 目的

本文档定义“校园互助服务平台”的功能性和非功能性需求。当前版本根据最新优先级调整，将 MVP 核心收缩为“可信代取服务闭环”：用户完成实名认证后，可发布和承接代取服务，并通过需求大厅、状态流转、完成凭证、支付沙箱和评价机制完成基础业务闭环。

### 1.2 文档约定

- **优先级定义（MoSCoW）：**
  - `[Must]`：必须实现，否则核心代取业务不可用
  - `[Should]`：应完成 MVP 基础实现，但不阻断核心闭环
  - `[Could]`：有时间再实现，不影响核心业务
  - `[Won't]`：本次 MVP 不涉及
- **需求编号规则：** `FR-[模块]-[序号]`（功能需求）、`NFR-[类别]-[序号]`（非功能需求）
- **来源标注：** `[AI生成]`、`[人工补充]`、`[AI生成+人工修改]`、`[人工修改]`

### 1.3 预期读者

- 产品与需求负责人：关注第 2 章和第 3 章
- 开发与测试人员：关注第 3 章、第 4 章和第 5 章
- 设计人员：参考功能边界和用户流程

### 1.4 产品范围

该产品是面向电脑端浏览器的校园互助 Web 平台。MVP 核心业务仅保留代取服务：用户管理、实名认证审核、代取需求大厅、代取发布与接单、完成凭证、确认完成、支付宝沙箱预付款/退款/结算、我的代取记录。评价体系和站内通知作为 Should 级基础能力。失物招领、二手交易、私聊、举报申诉等作为可选能力；论坛、找搭子、学习辅导、真实线上支付等不进入本次 MVP。

- IEEE Std 830-1998
- 问卷调研数据（50 份有效问卷）
- 用户访谈记录（6 人次）

## 2. 总体描述

### 2.1 产品视角

系统为独立运行的前后端分离 Web 应用。MVP 阶段不接入真实第三方支付生产环境，仅使用支付宝沙箱验证有报酬代取服务的预付款、退款和结算流程，不发生真实资金流转。

### 2.2 产品功能（高层概要）

1. **用户管理：** 用户名密码注册登录、个人资料编辑、密码安全存储。
2. **实名认证：** 用户提交学号、姓名和认证材料，管理员审核通过后方可发布、接单和评价。
3. **代取需求大厅：** 展示待接单代取服务，支持详情查看、校区筛选、报酬类型筛选和发布时间倒序。
4. **代取闭环：** 发布代取、接单、查看取件凭证、上传完成凭证、确认完成、取消和状态流转。
5. **我的代取记录：** 用户可分别查看“我的发布”和“我的接单”；两个分类下均提供总览和按代取状态分类查看。
6. **评价体系：** 代取完成后双方互评，计算并展示好评率。
7. **站内通知：** 预留通知能力，用于认证结果、接单、完成凭证、确认完成和评价等事件。

### 2.3 用户角色

| 用户角色 | 特征描述 | 主要行为 |
|---------|---------|---------|
| 访客 | 未登录或未认证用户 | 浏览公开的代取需求大厅 |
| 普通用户（学生） | 注册并通过实名认证的在校学生 | 发布代取服务、接单、上传凭证、确认完成、评价 |
| 管理员 | 负责实名认证审核的管理人员 | 审核用户实名认证材料 |

同一普通用户在不同服务中可同时作为发布方和接单方，两种身份由具体代取服务上下文决定，非互斥。

### 2.4 运行环境

- 客户端：Chrome、Edge、Safari 等主流桌面浏览器
- 服务器端：Linux 环境，后端服务，MySQL；Redis 仅作为可选优化

### 2.5 设计和实现限制

- MVP 阶段使用用户名密码注册登录，不接入运营商短信验证码。
- 每个学号只能绑定一个账号，已认证学号不可重复使用。
- 学号、姓名和认证材料仅用于实名认证和管理员审核，不向普通用户公开展示。
- 学院和联系方式为个人资料选填项；联系方式未填写时不展示。
- MVP 阶段不做普通发布内容人工审核；代取服务发布后通过基础字段校验和支付状态进入对应状态。
- MVP 阶段仅支持代取服务，不支持代购、代办或需要接单方垫付本金的服务。
- MVP 阶段不接入真实线上支付，不为二手交易提供平台担保或支付能力。

#### 代取服务状态说明

| 状态 | 含义 | 主要触发操作 |
|------|------|--------------|
| 待支付 | 有报酬代取服务已提交，等待发布者在 3 分钟内完成支付宝沙箱预付款 | 发布有报酬服务 |
| 待接单 | 服务可在代取需求大厅展示，等待其他认证用户接单 | 无报酬发布成功 / 有报酬支付成功 |
| 进行中 | 已有接单方承接服务，其他用户不可再接单 | 用户接单 |
| 已完成 | 发布方确认代取服务完成 | 发布方确认完成 |
| 已取消 | 服务被发布方取消、待支付超过 3 分钟未支付或接单截止超时取消 | 取消 / 待支付超时 / 接单截止超时 |

## 6. 附录

### 6.1 词汇表

| 术语 | 定义 |
|------|------|
| MVP | Minimum Viable Product，最小可行产品 |
| 代取服务 | 代取快递、外卖、已明确物品等，不要求接单方垫付本金的校园服务 |
| 代取需求大厅 | 展示待接单代取服务的公开列表页 |
| 有报酬代取 | 发布方通过支付宝沙箱预先支付服务报酬的代取服务 |
| 无报酬代取 | 不涉及资金流转的代取服务 |
| 好评率 | 基于用户收到的代取服务评价计算的可信度指标 |

### 6.2 当前 MVP 范围说明

**Must：** 用户管理、管理员审核实名认证、代取需求大厅、代取闭环、我的代取记录。  
**Should：** 评价体系、站内通知。  
**Could：** 关键词搜索、失物招领轻量功能、二手交易轻量功能、用户举报/封禁、评价/代取申诉、私聊、Redis 优化。  
**Won't：** 真实线上支付、二手担保支付、发布内容人工审核、论坛、找搭子、拼车、学习辅导、官方认证 API、多校区自动路由、智能推荐、数据分析、高并发扩容、微服务、事件驱动/MQ。
