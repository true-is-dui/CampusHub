# P4 演示说明

## 1. 运行方式

### 后端

```bash
cd backend
./mvnw.cmd spring-boot:run
```

默认地址：

- 后端接口基址：`http://localhost:8080/api`

运行前需要准备：

- MySQL 数据库
- `application-local.yaml` 或环境变量中的数据库连接配置
- `JWT_SECRET`

### 前端

```bash
cd frontend
npm install
npm run dev
```

默认地址：

- 前端页面：`http://localhost:5173`

## 2. 演示账号 / 测试数据

建议准备两类账号：

- 发布方账号：已实名认证通过
- 接单方账号：已实名认证通过

建议准备的测试数据：

- 1 条待接单代取需求
- 1 条进行中的代取需求
- 1 条已完成代取需求

若助教现场从头演示，可直接新建两名普通用户并走完整流程。

## 3. 核心演示路径

### 路径 A：完整主流程

1. 注册发布方账号并登录
2. 完善个人资料，提交实名认证
3. 管理员审核通过实名认证
4. 发布代取需求并上传取件凭证
5. 使用另一账号登录并接单
6. 接单方上传完成凭证
7. 发布方确认完成
8. 在“我的发布 / 我的接单”中查看状态变化

### 路径 B：异常流程展示

1. 未登录直接访问发布或接单接口，展示 `40101`
2. 以有偿订单缺少金额提交，展示 `40001`
3. 对同一需求重复接单，展示 `40901`

## 4. 演示重点

- JWT 登录态是否正确传递
- 实名认证前后操作权限是否正确
- 代取状态是否按 `WAITING_ACCEPT -> IN_PROGRESS -> COMPLETED` 流转
- 上传图片和受保护文件访问是否正常
- 异常提示是否清晰

## 5. 说明

本次 P4 演示以课程 MVP 为边界，不展示以下扩展内容：

- 失物招领
- 二手交易
- 私聊
- WebSocket
- Redis / MQ / 搜索优化
- 复杂管理后台

这样可以保证演示链路聚焦且稳定。
