# P4 Sprint 合并任务看板

## 1. 分工原则

P4 编码阶段采用“两名前端、两名后端/测试联调”的协作模式。团队在任务划分上以可信代取服务闭环为主线，优先保证 P0 核心功能稳定可运行，再完成 P1 评价与通知等增强能力的基础版本。

本阶段后端工作由后端核心开发负责人统一把关业务主链路和设计一致性，后端测试联调负责人同步参与接口验证、测试数据构造、异常流程复现和 Bug 修复记录。前端工作由前端基础与核心页面负责人先完成工程骨架、路由权限和核心页面，前端业务页面与体验负责人补齐我的记录、评价通知、表单校验和演示体验。

该分工既保证后端核心业务逻辑由熟悉前期设计文档的成员统一把关，也保证测试、联调和阶段文档有明确负责人。所有实现任务均应以 `docs/P3/api_design.yaml` 为接口契约，以 `docs/P3/database_design.md` 和 `docs/P3/class_design.md` 为后端实现边界，不新增未确认的业务模块、状态值、接口或数据库表。

本看板中的实名认证审核定位为“实名认证提交与基础审核接口 / 演示用审核入口”，不扩展为完整管理后台；积分相关能力定位为“平台内虚拟积分：认证赠送、每日签到、发布扣减、取消退回、完成转移的基础演示流程”，不扩展为生产级资金系统。积分变动与代取状态在服务层同一事务内完成；积分不足导致发布失败不写流水，由业务层返回错误。超时取消可以通过定时任务或手动触发验证完成，不阻塞 P0 核心闭环演示。

## 4. 风险与应对

| 风险 | 影响 | 应对措施 | 负责人 |
|------|------|----------|--------|
| 后端核心逻辑集中，开发压力较高 | 用户认证、实名审核、代取状态、积分、评价和通知集中在后端服务层，可能影响 P0 闭环进度。 | 优先保证 P0 核心闭环；P1 评价与通知先实现基础可用版本；毛嘉和 提前准备接口测试和 Bug 日志模板，及时复现联调问题。 | 谢易轩、毛嘉和 |
| 支付与订单状态流转耦合，边界容易出错 | 可能出现发布扣减、取消退回、确认完成转移与代取状态不一致。 | 代取状态与积分变动由后端服务层统一在同一事务内维护；积分流水只保存积分变动记录和追踪字段；积分不足时发布失败返回 409，不写流水；不扩展复杂支付仲裁或生产级资金清结算。 | 谢易轩、毛嘉和 |
| 前后端接口字段不一致 | 页面联调失败，可能出现请求体字段、响应字段、枚举值或分页结构不匹配。 | 所有接口以 `docs/P3/api_design.yaml` 为准；前端封装接口前核对路径、方法、字段、枚举和错误结构；联调问题明确区分 API 文档、后端实现、前端实现或测试数据问题。 | 毛嘉和、姚圳锴、张皓 |
| AI 生成代码可能不符合前期设计 | 可能引入未确认字段、状态、接口、表结构或过度设计组件。 | AI 代码必须人工审查；检查是否符合 P3 类设计、数据库设计和 API 设计；不得新增未确认字段、状态、接口或表结构；不得引入 WebSocket、消息队列、Redis 优化、真实生产支付等非当前 MVP 必做内容。 | 谢易轩、毛嘉和 |
| 测试和文档集中在后期导致遗漏 | 可能导致验收材料不完整，Bug 复现信息缺失，演示路径不稳定。 | 毛嘉和 提前准备接口测试集合和 Bug 日志模板；单元测试由谢易轩 主责、毛嘉和 协助；CI/CD 由谢易轩、毛嘉和 共同负责；README、演示说明和反思日志由全体按分工补齐。 | 全体成员 |

## 5. 阶段验收标准

- P0 优先级功能全部可用，包括用户注册登录、JWT 鉴权、个人资料、实名认证提交与基础审核、代取发布、浏览、接单、完成、取消、我的记录、文件上传和平台积分基础演示流程。
- P1 优先级功能基本可用，包括完成后双方互评、好评/中评/差评、差评理由必填、按发布方和接单方角色分别统计基础好评率、站内通知列表和已读/未读状态。
- 系统可以完整演示核心业务流程：注册或登录 -> 实名认证通过 -> 发布代取需求 -> 另一用户接单 -> 上传完成凭证 -> 发布方确认完成 -> 提交并展示评价 -> 查看相关通知。
- 核心模块单元测试覆盖率 >= 60%。
- 集成测试覆盖至少 1 个完整正常流程和 2 个异常流程。
- 已配置 CI/CD，能够自动执行依赖安装、基础检查、测试和构建。
- 有最近一次 CI/CD 运行结果记录。
- 有 AI 代码信任度实验报告，包含 Prompt、AI 直出结果、人工检查问题和修复前后对比。
- 有 AI 调试对决实验报告，至少记录 2 个 Bug 的人工调试与 AI 辅助调试对比数据。
- 有 Bug 修复日志，包含问题现象、根因分析、修复方案和验证结果。
- 有 README / 运行说明 / 演示说明，能够支持助教本地运行或按演示路径体验核心功能。
- 有 AI 协作反思日志 #4，内容覆盖本阶段 AI 使用清单、价值分析、误导分析、Prompt 改进和团队工程判断。
