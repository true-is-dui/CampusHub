## 3. 角色职责阐述

### 需求负责人 - 姚圳锴
- **对角色的理解**：
  需求负责人是用户与开发团队之间的桥梁，负责把用户的真实需求转化为团队可执行的任务。
- **核心职责与目标**：
1. 整理用户调研（问卷 + 访谈），收集真实需求
2. 编写需求规格说明书，确保需求完整、清晰
3. 管理需求变更，维护需求追踪矩阵
4. 目标：让团队做正确的事，避免开发出没人用的功能，确保项目实用性

### 架构负责人 - 张皓
- **对角色的理解**：架构负责人是项目的技术骨架搭建者，负责在需求与实现之间建立可靠、高效、可扩展的技术方案，为团队提供清晰的技术路线与决策依据。
- **核心职责与目标**：
1. 基于业务需求进行技术选型与架构设计，定义系统整体结构、模块划分与接口规范
2. 撰写架构决策记录（ADR），沉淀关键技术决策的背景、方案与权衡，确保团队技术认知一致
3. 制定技术规范与质量标准，主导关键技术难点的预研与方案验证，规避技术风险
4. 搭建稳定、可扩展、易维护的系统架构，保障项目长期演进能力，支撑业务目标高效达成

### 开发负责人 - 谢易轩
- **对角色的理解**：
  开发负责人是将需求和架构方案落地为实际系统的核心执行者，负责推动核心功能实现，并保证代码质量与开发规范。
- **核心职责与目标**：
1. 根据需求和设计方案完成核心功能开发，推动项目按计划实施
2. 制定并维护编码规范，保证代码风格统一、结构清晰
3. 负责代码审查与质量把关，及时发现并修复实现中的问题
4. 目标：确保系统实现稳定、规范，提高团队开发效率和代码可维护性

### 测试负责人 - 毛嘉和
- **对角色的理解**：
  测试负责人是质量保障体系的核心驱动者，需从全局视角规划测试活动，最终对产品交付质量负责。
- **核心职责与目标**：
1. 制定测试策略，根据项目特点与资源约束，设计分层测试方案，明确测试范围、方法与准入准出标准
2. 设计测试用例，主导用例体系构建，确保覆盖核心功能、边界条件、异常场景及非功能需求，并持续维护用例的复用性与可追溯性
3. 质量保障与过程改进，监控测试执行与缺陷趋势，分析质量短板，推动问题闭环
4. 风险管控与决策支持，评估发布风险，为版本发布提供可靠依据
5. 团队协同，协调测试资源，对齐开发、产品等角色预期，确保质量
